## Design Rationale

### Why Soda as a Modular Provider?

汽水音乐是中国最大的短视频平台旗下音乐服务，其 API 独特之处：
- 需要客户端签名 (BDMS/BDTicket)
- 依赖本机 Chrome Cookie
- 使用加密流 (decodedKey)
- 需要多 body 尝试才能解析播放地址

将这些逻辑从 `server.js` 提取为独立模块，使得：
- 其他 Provider (QQ/网易云) 可以复用相同设计模式
- 测试时可以单独验证每个模块
- 后续可以替换/升级签名算法

### Why format=auto?

设计的核心原则：**转码是有损不可逆操作**。

```
format=auto 行为:
- 源 FLAC → 保存 FLAC (无损保留)
- 源 AAC  → 保存 M4A  (保留编码)
- 源 MP3  → 保存 MP3  (无需转码)
- 加密流  → MP3       (必须转码)
```

如果 AI 音乐库需要原始质量做分析，format=auto 是必要条件。

### Why Asset Enhancement?

传统下载只保存音频文件，丢失大量元数据。设计目标：

```
每个下载文件都是一条完整的音乐记录:
├── Audio           → 可播放
├── metadata.json   → 机器可读的结构化数据
├── ID3 tags        → 播放器可直接读取
├── Cover art       → 可视化展示
└── Lyrics (.lrc)   → 歌词同步
```

这为后续的本地音乐库、Embedding、推荐系统打基础。

## Data Flow

```
用户操作                         后端处理
─────────                      ─────────
搜索歌曲  ──→  /api/soda/search  ──→ soda-api-client
播放歌曲  ──→  /api/soda/song/url ──→ soda-provider
                                        ↓
                                   soda-playback-resolver
                                        ↓
                                   Cookie + 签名 + 多body尝试
                                        ↓
                                   返回音频URL/Session

点击下载  ──→  /api/download/start ──→ download-manager
                                        ↓
                                   resolveTrackUrlForDownload
                                        ↓
                                   getDownloadCover + getDownloadLyrics
                                        ↓
                                   download-service.execute()
                                        ├── HTTP直接保存 (format=auto,无加密)
                                        └── FFmpeg转码 (加密流/格式转换)
                                        ↓
                                   download-asset.enhanceDownload()
                                        ├── metadata.json
                                        ├── node-id3 (ID3 tags)
                                        ├── Cover HTTP下载
                                        └── Lyrics写入
                                        ↓
                                   ~/Music/Mineradio/{Source}/
```

## Key Decisions

| Decision | Why |
|----------|-----|
| 依赖注入 (`setup()`) | 避免模块间循环依赖，方便测试 |
| Soda 使用 Map 存 Session | 比对象更易管理，自动GC |
| FFmpeg 用 spawn 而非 exec | 处理大文件不阻塞，流式处理 |
| windowsHide: true | 避免用户看到命令行窗口 |
| EXDEV fallback (copy+delete) | Windows 跨驱动器 rename 失败 |
| 并发限制 3 | 避免 FFmpeg 进程吃满CPU |
| cover/lyrics 自动获取 | 前端不需要传 coverUrl/lyricUrl |
| 前端也传 coverUrl | 双保险：快速路径 + 后端fallback |

## Known Limitations

1. **Soda 依赖本地浏览器** — Cookie 过期后需重新登录
2. **加密流依赖本机解码器** — 无解码器时降级为低音质
3. **QQ/网易云下载链路未 E2E 验证** — 只有 Soda 经过了完整测试
4. **musicDir is server-side only** — 下载路径没有 UI 配置
5. **下载队列不支持暂停** — 只支持取消
6. **没有下载历史管理 UI** — 只能通过 `/api/download/list` 查看
7. **node-id3 只支持 MP3** — FLAC 不自带 ID3，metadata.json 是唯一结构化记录
